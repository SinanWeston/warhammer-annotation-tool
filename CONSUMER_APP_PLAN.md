# Plan: Warhammer 40K Miniature Scanner — Consumer PWA

## Context

The photoanalyzer project currently has an **annotation tool** (React + Express) for creating YOLO training data. The end goal is a consumer app where users photograph a Warhammer 40K tabletop game and the app identifies/counts miniatures by faction. While the user annotates training data, we'll build the consumer-facing frontend as a PWA.

**MVP scope**: Point camera or upload photo → detect miniatures → show identified units grouped by faction with counts.

**User answers**:
- Core feature (MVP): Identify & count units by faction
- Input: Built-in camera + photo upload
- Tech: PWA (works on iOS, Windows, Linux, macOS)
- Theme: Dark 40K themed (reuse existing gothic assets)

## Architecture Decision: Separate App in Monorepo

Create `consumer/` as a new workspace alongside `frontend/` (annotation tool). Reasons:
- Consumer app has totally different UX (camera → results) vs annotation tool (bbox drawing)
- Separate PWA config, service worker, manifest, icons
- Independent deployment — consumer app can be hosted on its own domain
- Smaller bundle — no annotation code shipped to end users
- Shared Tailwind theme config via copy

## Sprint 2: Scan Page — Camera + Upload

### `App.tsx`

Simple state-based routing: `'scan' | 'results'`. Passes detection results from scan to results page. Dark background, centered layout, max-width container.

### `ScanPage.tsx`

Two-section layout:
1. **Camera section** — Large viewfinder area with shutter button
2. **Upload section** — "or upload a photo" with file picker

On capture/upload → calls `POST /api/detect` → navigates to results.

### `CameraCapture.tsx`

Uses `navigator.mediaDevices.getUserMedia()`:
- Rear camera preferred (`facingMode: 'environment'`)
- Live video preview with gothic border + scan line overlay
- Large circular shutter button (pulsing glow animation)
- Captures frame to canvas → converts to blob
- Camera permission handling with fallback message
- Front/rear camera switching

### `PhotoUpload.tsx`

- Drag-and-drop zone with dashed border
- Click to open file picker
- Accepts `.jpg`, `.jpeg`, `.png`, `.webp`
- Shows image preview after selection
- "Scan" button to submit

### `ScanAnimation.tsx`

Full-screen overlay shown during inference:
- Pulsing icon with orbiting ring (CSS animation)
- Cycling status text ("Analyzing battlefield...", "Identifying factions...", etc.)
- Indeterminate progress bar
- Dark semi-transparent backdrop

## Sprint 3: Results Page — Detection Display

### `ResultsPage.tsx`

Receives detection results. Layout:
1. **Scanned image** — with detection overlay (bboxes drawn on canvas with labels)
2. **Inference time** — small footer text
3. **Army Summary** — total units detected, grouped by faction
4. **Faction cards** — one per detected faction with unit count and confidence
5. **Action bar** — "Scan Another" button

### `ArmySummary.tsx`

Top-level summary:
- Total miniatures detected (large number with glow)
- Number of factions identified
- Grid of faction badges with counts (color-coded)

### `ResultsCard.tsx`

Per-faction card:
- Faction name in gothic font
- Faction-specific accent color
- Unit count (large)
- Average confidence percentage
- Expandable detail: list of individual detections with confidence bars

### `detection.ts`

```typescript
interface DetectionResult {
  imageWidth: number
  imageHeight: number
  detections: Detection[]
  summary: FactionSummary[]
  totalDetected: number
  inferenceTimeMs: number
}

interface Detection {
  faction: string
  confidence: number
  bbox: { x: number; y: number; width: number; height: number }
}

interface FactionSummary {
  faction: string
  count: number
  avgConfidence: number
  color: string
}
```

## Sprint 4: Backend Detection Endpoint

### Modified: `backend/src/index.ts`

Added endpoint: `POST /api/detect`
- Accepts multipart image upload (multer)
- Calls `detectAndSummarize()` from YOLO inference service
- Returns `DetectionResult` JSON
- Cleans up uploaded file after processing

### Modified: `backend/src/services/yoloInferenceService.ts`

Added:
- `detectAndSummarize(imagePath)` — Runs YOLO inference, groups by faction, computes stats
- `FACTION_COLORS` lookup map for 15 factions
- `ConsumerDetection`, `FactionSummary`, `DetectionResult` interfaces

## Running the Consumer App

```bash
# From repo root
npm run dev:consumer    # Starts on http://localhost:5174
npm run dev:backend     # Backend must be running for /api/detect

# Or run everything
npm run dev             # Starts frontend + backend (annotation tool)
npm run dev:consumer    # In another terminal for consumer app
```
